## 5. 辯證分析 (Dialectic Analysis)

### 正題 (Thesis)

4/5 non-preamble elements MAPPED (mandatory)。HE frame structure、channel estimation on training fields、beam-change indicator decoding 全部在標準中有明確 mandatory 對應。但 Claim 1 的核心創新 [1E]——接收端利用 beam-change=0 的訊號，結合 L-LTF、HE-LTF 和 re-modulated signal fields 來推導 enhanced channel response matrix——在 802.11ax-2021 全文（767 頁）搜索後確認不存在任何 normative 要求。**NOT ESSENTIAL。**

### 反題 (Antithesis)

**MTK/Sisvel 可能主張 #1**: Beam-change indicator 的「目的」就是為了讓 receiver 做 channel estimation enhancement，因此任何 compliant 實作「必然」會利用此 indicator 來 combine training fields。

→ 駁回：此論點混淆了「標準提供機制」與「標準要求使用方式」。802.11ax 定義了 BEAM_CHANGE 的 TX-side signaling，但 receiver 如何使用此資訊完全是 implementation choice。Receiver 可以忽略 BEAM_CHANGE=0 而僅基於 HE-LTF 做 channel estimation，仍完全 compliant。

**MTK/Sisvel 可能主張 #2**: Table 27-18 的 Beam Change=0 定義「indicates that the pre-HE modulated fields are spatially mapped the same way as the first symbol of the HE-LTF on each subcarrier」——此定義暗示 receiver 應利用此一致性。

→ 駁回：此定義是對 TX-side spatial mapping 的描述，不是 RX-side processing mandate。"indicates" 是 informational description，不是 "the receiver shall combine"。

**MTK/Sisvel 可能主張 #3**: Equations (27-8), (27-10), (27-13), (27-15), (27-17), (27-19) 定義了 BEAM_CHANGE=0 時的 spatial mapping（pre-HE 和 HE fields 使用相同 steering matrix）——這些方程式支持 combining 的技術基礎。

→ 駁回：這些方程式定義的是 TX-side waveform generation，確認了在 BEAM_CHANGE=0 時 spatial mapping 一致。但標準仍未規範 RX-side 要如何利用此一致性來 enhance channel estimation。TX-side 的數學定義 ≠ RX-side 的 processing mandate。

### 合題 (Synthesis)

結論 NOT ESSENTIAL，信心 **High**。標準的 normative text 在 TX-side 完整定義了 BEAM_CHANGE signaling 和 spatial mapping equations，但在 RX-side 僅定義了 HE-LTF 用於 channel estimation（§27.3.11.10），從未 mandate 接收端結合 L-LTF + HE-LTF + re-modulated signal fields 推導 enhanced channel response matrix。專利的核心創新在於 RX-side 的 channel estimation enhancement algorithm，這是 802.11ax 標準刻意不規範的 implementation domain。

**Known Unknowns**:

1. 802.11-2020 base standard（非 amendment）中是否有對 L-LTF-based channel estimation 的更具體 normative 要求——本分析僅搜索了 802.11ax-2021 amendment 文件
2. Wi-Fi Alliance certification test plan 是否要求特定的 channel estimation performance level 而間接 mandate 此 technique——但 certification test plan 不等於標準規範

## 6. 戰略意涵 (Strategic Implications)

**Cui Bono?** MediaTek 透過 Sisvel Wi-Fi 6 pool 授權此專利。Pool 收費模式依賴 SEP 數量的 portfolio 效應。即使單件專利 essentiality 可被挑戰，在 pool 架構下被告通常接受 bundled rate 而非逐件抗辯。MTK 宣告此專利為 essential 的動機是擴大其在 Sisvel pool 中的 contribution weight。

**Declaration Status**: Declared via Sisvel Wi-Fi 6 pool。若判定 NOT ESSENTIAL，此專利屬於 over-declaration 案例。

**Key Gap**: Element [1E] — channel estimation enhancement algorithm 是 implementation-level optimization，非 standard mandate。標準定義了 BEAM_CHANGE signaling（TX-side），但不規範 receiver 如何利用此訊號。
